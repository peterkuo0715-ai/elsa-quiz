#### [F-017] 商家 Dashboard 重設計
- **優先級**：P0
- **目標版本**：v1.1
- **狀態**：✅ 已完成
- **需求描述**：
  重新設計商家後台 Dashboard，參考頂級 SaaS 帳務系統（Stripe Dashboard / Shopify Balance）布局。
  分 5 層：

  **第一層：核心數字 Hero 區**

  **第二層：預計入帳（新功能）**

    - 已出貨未到貨：筆數 + 金額
    - 已付款未出貨：筆數 + 金額
  - 計算邏輯：查詢 settlement_items 狀態為 IN_APPRECIATION_PERIOD / SHIPPED / PAID 的 netAmount 加總

  **第三層：趨勢圖**
  - 近 7 天 / 30 天雙線折線圖
  - 綠線：每日入帳（settlement credit 加總）
  - 紅線：每日扣款（refund + commission + dispute debit 加總）
  - 可切換 7 天 / 30 天視圖

  **第四層：快速行動 + 待處理**
  - 「立即提領」快捷鍵
  - 待處理爭議案件數
  - 負餘額警示
  - 銀行帳號審核狀態

  **第五層：最近帳務事件 Timeline**
  - 最近 10 筆 ledger entries

- **驗收標準**：
  1. 可提領餘額為最醒目的數字
  2. 預計入帳顯示三層明細且金額正確（= settlement_items 未結算的 netAmount 加總）
  3. 趨勢圖可切換 7/30 天，雙線正確
  4. 待處理區顯示正確數量
  5. 頁面載入 < 1 秒

### 八、業務規則補充

#### [R-001] 抽成金額無條件進位至整數元
- **優先級**：P0
- **狀態**：✅ 已完成
- **規則描述**：
  平台抽成金額若計算結果為小數，一律**無條件進位（ceiling）到整數元**。
  此規則確保平台抽成不會因四捨五入而少收。
- **計算範例**：
  - 商品未稅價 NT$285，抽成率 10% → 285 × 0.1 = 28.5 → **進位為 NT$29**
  - 商品未稅價 NT$1,000，抽成率 10% → 100.0 → **NT$100（整數不變）**
  - 商品未稅價 NT$333，抽成率 12% → 39.96 → **進位為 NT$40**
- **影響範圍**：
  - `SettlementService` 結算時的抽成計算
  - `RefundService` 退款時的抽成退還計算（退還時也用 ceiling）
  - Simulator 中的抽成計算
  - 所有顯示抽成金額的 UI
- **技術實作**：
  - `money.ts` 新增 `moneyCeil()` 函式（無條件進位到整數）
  - 抽成計算改用 `moneyCeil()` 取代 `moneyRound()`
